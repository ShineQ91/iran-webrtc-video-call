# اتاق ویدیویی ایران 🇮🇷

یک پلتفرم کاملاً رایگان برای تماس صوتی و تصویری گروهی، بهینه شده برای اینترنت ضعیف و موبایل. همه کاربران مستقیماً به یک اتاق عمومی به نام "ایران" متصل می‌شوند.

## ✨ ویژگی‌ها

- 🆓 **کاملاً رایگان** - بدون نیاز به API یا سرور پولی
- 📱 **موبایل فرندلی** - PWA با تجربه اپلیکیشن نیتیو
- 🌐 **اتاق عمومی ایران** - همه کاربران در یک اتاق مشترک
- 🚀 **بهینه برای اینترنت ضعیف** - کار می‌کند حتی زیر 500-700 کیلوبیت
- 🔒 **امن** - رمزنگاری سرتاسر WebRTC
- 💬 **چت متنی** - ارتباط متنی همزمان با تماس
- 📹 **کنترل کامل** - روشن/خاموش کردن دوربین و میکروفون
- 🖥️ **اشتراک صفحه** - امکان به اشتراک گذاشتن صفحه نمایش

## 🛠️ تکنولوژی‌ها

### بک‌اند
- **Node.js** - سرور جاوااسکریپت
- **Socket.IO** - سیگنالینگ بلادرنگ
- **Express** - وب سرور سبک

### فرانت‌اند
- **WebRTC** - ارتباط مستقیم بین کاربران
- **HTML5/CSS3** - رابط کاربری مدرن
- **Tailwind CSS** - استایل‌دهی سریع
- **JavaScript Vanilla** - بدون فریمورک سنگین

### بهینه‌سازی
- **Adaptive Bitrate** - تنظیم خودکار کیفیت
- **VP8/H.264** - کدک‌های ویدیویی بهینه
- **Opus** - کدک صوتی برای پهنای باند کم
- **PWA** - تجربه اپلیکیشن روی موبایل

## 🚀 راه‌اندازی سریع

### پیش‌نیازها
- Node.js نسخه 16 یا بالاتر
- npm یا yarn

### نصب و اجرا

1. **کلون کردن ریپازیتوری**
```bash
git clone https://github.com/your-username/iran-webrtc-video-call.git
cd iran-webrtc-video-call
```

2. **نصب وابستگی‌ها**
```bash
npm install
```

3. **اجرای سرور**
```bash
npm start
```

4. **باز کردن مرورگر**
```
http://localhost:3000
```

### حالت توسعه
```bash
npm run dev
```

## 📱 استفاده روی موبایل

1. مرورگر گوشی را باز کنید
2. آدرس سرور را وارد کنید
3. روی دکمه "ورود به اتاق ایران" کلیک کنید
4. اجازه دسترسی به دوربین و میکروفون را بدهید
5. از تماس گروهی لذت ببرید!

## 🔧 تنظیمات پیشرفته

### تغییر پورت سرور
```bash
PORT=8080 npm start
```

### اضافه کردن سرور TURN سفارشی
در فایل `public/app.js`، آرایه `iceServers` را ویرایش کنید:

```javascript
this.iceServers = [
    { urls: 'stun:stun.l.google.com:19302' },
    {
        urls: 'turn:your-turn-server.com:3478',
        username: 'your-username',
        credential: 'your-password'
    }
];
```

## 🌐 دیپلوی

### روی Render.com

1. **پوش کردن به گیت‌هاب**
```bash
git add .
git commit -m "Initial commit"
git push origin main
```

2. **ساخت وب سرویس جدید**
- به [Render.com](https://render.com) بروید
- ریپازیتوری گیت‌هاب را متصل کنید
- `Node.js` را به عنوان محیط اجرا انتخاب کنید
- دستور Build: `npm install`
- دستور Start: `npm start`

3. **تنظیمات پیشرفته**
- پورت: `3000`
- متغیرهای محیطی: `NODE_ENV=production`

### روی VPS شخصی

1. **نصب Node.js**
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

2. **استفاده از PM2**
```bash
npm install -g pm2
pm2 start server.js --name "iran-video-call"
pm2 startup
pm2 save
```

3. **استفاده از Nginx (اختیاری)**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 🔍 عیب‌یابی

### مشکلات رایج

**1. اتصال برقرار نمی‌شود**
- بررسی فایروال و پورت‌ها
- اطمینان از HTTPS در محیط production
- تست سرورهای STUN/TURN

**2. کیفیت ویدیو پایین است**
- بررسی پهنای باند اینترنت
- تنظیم رزولوشن در `mediaConstraints`
- فعال بودن Adaptive Bitrate

**3. صدا کار نمی‌کند**
- بررسی دسترسی به میکروفون
- تنظیمات مرورگر برای میکروفون
- تست با هندزفری

**4. روی موبایل کار نمی‌کند**
- بررسی HTTPS
- تست با مرورگرهای مختلف
- بررسی دسترسی به دوربین/میکروفون

### لاگ‌ها
برای دیباگ کردن، کنسول مرورگر را باز کنید و لاگ‌ها را بررسی کنید:

```javascript
// در کنسول مرورگر
localStorage.debug = '*';
```

## 🚨 مشکلات خاص ایران

### فیلترینگ و NAT
- استفاده از سرورهای STUN متعدد
- پیکربندی سرور TURN در صورت نیاز
- تست با اپراتورهای مختلف

### راه‌حل‌ها
1. **سرورهای چندگانه STUN**: بهبود اتصال در شرایط مختلف NAT
2. **TURN فال‌بک**: برای مواردی که P2P ممکن نیست
3. **اتصال مجدد خودکار**: مدیریت قطعی‌های موقت
